 18def convCircle(singleChannelFrame, radius, radiusTolerance=None, offscreenParticles=False, kernelBlurKernel=3, outlineOnly=False, outlineThickness=.05, negativeHalo=False, haloThickness=.03, negativeInside=False, peakDownsample=10, minPeakPrevalence=.1, intensitySoftmax=1.2, intensitySoftmin=.1, invert=False, allowOverlap=True, fitPeaks=True, debug=False):
 19    """
 20    Perform convolution circle detection on the provided image.
 21
 22    A brief description of the technique is described below; for
 23    a more comprehensive discussion, see [1].
 24
 25    The convolution method identifies circles in an image by convolving
 26    a kernel (representing what a particle should look like) with the
 27    image within which you would like to detect circles. This essentially
 28    finds the likelihood of any particle location being the center for
 29    such an object. By performing peak detection on this 2D quantity,
 30    we can identify the most likely places for particles to be.
 31
 32    While this technique can be used for non-circular objects, this 
 33    implementation makes use of several simplifications/approximations
 34    that make it suitable only for radially symmetric objects.
 35
 36    Tends to give more accurate and consistent results than the Hough
 37    technique (see `pepe.tracking.houghCircle()`), though evaluation
 38    times can be longer. When the radius of the circle is not well known
 39    (with no initial guess), the Hough method should be used. That being said,
 40    this implementation can fine tune the provided radius, but this requires
 41    the provided value to be reasonably close to the actual value (within 
 42    ~10 pixels is ideal).
 43
 44    The ability to get subpixel precision by interpolating/fitting curves to the
 45    convolution is currently WIP, and currently the method can, at best, give results
 46    accurate to within a single pixel.
 47
 48    
 49    Parameters
 50    ----------
 51 
 52    singleChannelFrame : np.uint8[H, W]
 53        The frame to perform circle detection on. Should not include multiple
 54        color channels. If it does, the channels will be averaged to form a
 55        grayscale image.
 56
 57    radius : float or [float, float]
 58        The radius to look for, or min and max radii to constrain the circle
 59        detection. If a single value is provided, a range will be constructed
 60        around that value with the `radiusTolerance` parameter.
 61
 62        Note that this method is more sensitive to the initial guess of radius
 63        than `pepe.tracking.houghCircle()`. Ideally, the radius should be within
 64        ~10 pixels of the true radius (or a range of radii should only be ~15 pixels wide).
 65
 66        NOTE: Varying the radius is still under development, and likely does not work that great.
 67
 68    radiusTolerance : float or None
 69        The half-width of the range of radii that will be tested for detecting particles.
 70        ie. the range will be calculated as `[radius + radiusTolerance, radius - radiusTolerance]`.
 71
 72        If set to `None`, radius will be fixed at whatever value is provided via `radius` argument.
 73
 74        NOTE: Varying the radius is still under development, and likely does not work that great.
 75
 76    offscreenParticles : bool
 77        Whether or not the algorithm should look for particles whose centers are outside of
 78        the image domain (but some parts of the particles still show up in the image).

 80        This expands the padding provided to the fft/ifft calculations, so as to avoid cutting
 81        off what might be a potential particle. As a result, a value of True will increase computation times.

 97

100

102        of pixels.
103
104    negativeInside : bool
105        Whether to fill the inside of the particle mask with a negative value, which 
106        further promotes identifying outlines of circles.
107

109
110    peakDownsample : int
111        The factor by which to downsample the image when performing the initial peak
112        detection. The final peak detection will always be performed at the original
113        resolution, so this should not affect the accuracy of the detected circles,
114        though it is possible that some particles in a densely packed system may
115        not be detected. False
116
117        Set to `1` or `None` to perform no downsampling, though this is not recommended,
118        as the peak finding algorithm is O(N) where N is the total number of grid points.
119
120    minPeakPrevalence : float
121        The minimum prevalence (or persistence) of peaks that will be considered as
122        particles. Should be a float in the range `[0, 1.]`, representing the percent
123        of the total intensity data range that must be spanned by a feature.
124
125        See `pepe.topology.findPeaks2D()` for more information.
126
127    fitPeaks : bool
128        Whether or not to fit each peak in the convolution with a lorentzian
129        form and use that center as the location of the particle (instead of
130        the simple maximum point).
131
132    intensitySoftmax : float
133        Value at which to cap the intensity values in the image, calculated as
134        this value multipled by the mean of the image.
135
136        Set to `None` to skip this preprocessing step.
137
138    intensitySoftmin : float
139        Zero-out all values that have an intensity below this factor times the mean of
140        the image.
141

143
144    allowOverlap : bool
145        Whether or not to allow detections of particles that overlap. If set to
146        False, then the particle with the stronger signature will be kept
147        
148    debug : bool
149        Whether or not to draw debug information on a plot.

151    invert : bool

153        image (and convolution peak finding will go from looking for maxima
154        to looking for minima).
155

157    -------
158
159    centers : np.ndarray[N,2]
160        The detected circles' centers.
161
162    radii : np.ndarray[N]
163        The detected circles' radii.
164
165
166    References
167    ----------
168
169    [1] Franklin, S. V., & Shattuck, M. D. (Eds.). (2016). Handbook
170    of Granular Materials. Chapter 2: Experimental Techniques. p.63-68.
171    CRC Press. https://doi.org/10.1201/b19291

174    # General housekeeping of arguments, see if we need to parse anything
175    if offscreenParticles:
176        paddingFactor = 2.
177    else:
178        paddingFactor = 1.1
179
180    if peakDownsample is None:
181        peakDownsample = 1
182 
183    # Check if we are given a range of radii or just a single value
184    possibleRadii = None
185
186    if hasattr(radius, '__iter__'):
187        if len(radius) == 2 and radius[0] < radius[1]:
188           possibleRadii = np.arange(radius[0], radius[1]+1)
189        else:
190            raise Exception(f'Invalid radius provided to convCircle()!\n Expected scalar or [min, max], but got {radius}!')
191    elif radiusTolerance is not None:
192        # Use the tolerance
193        possibleRadii = np.arange(radius - radiusTolerance, radius + radiusTolerance)
194    else:
195        # Just the single value
196        possibleRadii = np.array([radius])
197
198    # Start with just the mean of the possible radii options
199    initialRadius = np.mean(possibleRadii)
200
201    if singleChannelFrame.ndim > 2:
202        imageArr = np.mean(singleChannelFrame, axis=-1)
203    else:
204        imageArr = np.copy(singleChannelFrame)
205            
206    if invert:
207        imageArr = np.max(imageArr) - imageArr
208
209    if intensitySoftmax is not None:
210        softmax = np.mean(imageArr) * intensitySoftmax
211        if not invert:
212            imageArr[imageArr >= softmax] = softmax
213        #else:
214        #    imageArr[255 - imageArr >= softmax] = softmax
215
216    if intensitySoftmin is not None:
217        softmin = np.mean(imageArr) * intensitySoftmin
218        if not invert:
219            imageArr[imageArr <= softmin] = 0
220        #else:
221        #    imageArr[255 - imageArr <= softmin] = 0
222
223    # Calculate convolution
224    convArr = circularKernelFind(imageArr, initialRadius, fftPadding=int(initialRadius*paddingFactor),
225                                 outlineOnly=outlineOnly, outlineThickness=outlineThickness,
226                                 negativeHalo=negativeHalo, haloThickness=haloThickness,
227                                 negativeInside=negativeInside, kernelBlurKernel=kernelBlurKernel, debug=debug)
228
229    if debug:
230        plt.show()
231
232    # Downsample data by 5-10x and run peak detection
233    downsampledConvArr = cv2.resize(cv2.blur(convArr, (peakDownsample,peakDownsample)), (0,0),
234                                    fx=1/peakDownsample, fy=1/peakDownsample, interpolation=cv2.INTER_CUBIC)
235
236    peakPositions, peakPrevalences = findPeaks2D(downsampledConvArr, minPeakPrevalence=minPeakPrevalence)
237
238    lorentzPositions = np.zeros((len(peakPositions), 2))
239    cleanedConvArr = np.zeros_like(convArr)
240
241    # We have the option to fit lorentzian peaks to each maximum in the convolution
242    # matrix, or we can just take the peak positions as they are
243    if fitPeaks:
244        def objectiveFunction(params, localImage):
245            # Create a small grid across the area
246            Y,X = np.ogrid[:localImage.shape[0], :localImage.shape[1]]
247            Y += int(params["center_y"] - localImage.shape[0]/2)
248            X += int(params["center_x"] - localImage.shape[1]/2)
249
250            lorentzArr = lorentzian([Y,X], [params["center_y"], params["center_x"]], params["width"], params["amp"], params["offset"])
251            
252            return np.sum((localImage - lorentzArr)**2)
253
254        # Now we fit an appropriate function to each peak to see if we can refine the center
255        for i in range(len(peakPositions)):
256            upsampledPosition = np.array([peakPositions[i][0]*peakDownsample, peakPositions[i][1]*peakDownsample])
257            imagePadding = int(np.mean(possibleRadii))
258            # Crop out the small area of the image around the peak
259            localConv = convArr[max(upsampledPosition[0]-imagePadding, 0):min(upsampledPosition[0]+imagePadding,convArr.shape[0]-1), max(upsampledPosition[1]-imagePadding, 0):min(upsampledPosition[1]+imagePadding, convArr.shape[1]-1)]
260
261
262            #plt.imshow(localConv)
263            #plt.show()
264            # Now setup a fit to this function
265            params = Parameters()
266
267            params.add('center_y', value=localConv.shape[0]/2, vary=True)
268            params.add('center_x', value=localConv.shape[1]/2, vary=True)
269            params.add('width', value=imagePadding/2, vary=False)
270            params.add('amp', value=np.max(localConv)-np.min(localConv), vary=False)
271            params.add('offset', value=np.min(localConv), vary=False)
272        
273            result = minimize(objectiveFunction, params, args=[localConv], method='nelder', max_nfev=10000)
274            
275            if result is not None:
276                lorentzPositions[i,0] = upsampledPosition[0] - localConv.shape[0]/2 + result.params["center_y"]
277                lorentzPositions[i,1] = upsampledPosition[1] - localConv.shape[1]/2 + result.params["center_x"]
278
279                Y,X = np.ogrid[:convArr.shape[0], :convArr.shape[1]]
280                cleanedConvArr += lorentzian([Y,X], lorentzPositions[i], result.params["width"], result.params["amp"], result.params["offset"])
281
282            else:
283                lorentzPositions[i] = upsampledPosition
284
285    else:
286        for i in range(len(peakPositions)):
287            upsampledPosition = np.array([peakPositions[i][0]*peakDownsample, peakPositions[i][1]*peakDownsample])
288            lorentzPositions[i] = upsampledPosition
289
290    # Look around each peak to find the real peak in the full-resolution image
291    # (only if we are actually downsampling)
292    refinedPeakPositions = []
293    refinedRadii = []
294
295    if peakDownsample > 1:
296        # To do this, we use a non-linear optimization scheme
297        for i in range(len(peakPositions)):
298
299            costArr = []
300
301            def refineObjectiveFunction(params):
302                kernelArr = genCircularKernel(np.array([params["y"].value, params["x"].value]), params["r"].value,
303                                              imageArr.shape, kernelBlurKernel, outlineOnly,
304                                              outlineThickness, negativeHalo, haloThickness, negativeInside)
305
306                # We are minimizing this function, so we need the cost to be negative
307                # (unless we have to invert the image)
308                costArr.append((np.sum(kernelArr * imageArr) / np.sum(kernelArr)) * (int(invert)*2 - 1))
309                return (np.sum(kernelArr * imageArr) / np.sqrt(np.sum(kernelArr))) * (int(invert)*2 - 1)
310
311            params = Parameters()
312
313            params.add('y', value=lorentzPositions[i][0], min=lorentzPositions[i][0]-imageArr.shape[0]*.02, max=lorentzPositions[i][0]+imageArr.shape[0]*.02)
314            params.add('x', value=lorentzPositions[i][1], min=lorentzPositions[i][1]-imageArr.shape[1]*.02, max=lorentzPositions[i][1]+imageArr.shape[1]*.02)
315            # May or may not want to vary the radius
316            # Small tolerances on the radius are so that min != max
317            params.add('r', value=np.mean(possibleRadii), vary=(len(possibleRadii) > 1), max=np.max(possibleRadii)-.01, min=np.min(possibleRadii)+.01)
318
319            result = minimize(refineObjectiveFunction, params, method='powell', max_nfev=10000, options={"ftol": 1e-5, "xtol": 1e-5})
320
321            #print(fit_report(result))
322
323            if result is not None:
324                refinedPeakPositions.append([result.params["y"].value, result.params["x"].value])
325                refinedRadii.append(result.params["r"].value)
326                
327                #if debug:
328                #    plt.plot(costArr)
329                #    plt.show()
330
331            else:
332                refinedPeakPositions.append(lorentzPositions[i])
333                refinedRadii.append(np.mean(possibleRadii))
334
335    else:
336        refinedPeakPositions = lorentzPositions
337        refinedRadii = np.repeat(np.mean(possibleRadii), len(refinedPeakPositions))
338
339
340    # Now we (optionally) remove overlapping particles
341    # TODO: Allow overlap to be a float value, which allows overlaps
342    # that do not exceed a certain threshold (eg. that fraction of the radius).
343    if not allowOverlap:
344        # Calculate the contact matrix
345        # negative contact padding because we don't want to remove particles that are
346        # good but just close to each other
347        adjMat = adjacencyMatrix(refinedPeakPositions, refinedRadii, contactPadding=-int(np.mean(refinedRadii)/10)) - np.eye(len(refinedPeakPositions))
348
349        # Locate all of the particles that are overlapping
350        overlappingParticles = np.array([i for i in range(len(refinedPeakPositions)) if np.sum(adjMat[i,:]) > 0])
351
352        # Now find how good of detections each of these is, and order them from worst to best.
353        particleScores = np.zeros(len(overlappingParticles))
354        for i in range(len(particleScores)):
355            # Need to convert to int16 since we might have negative values
356            pMask = genCircularKernel(refinedPeakPositions[i], refinedRadii[i],
357                                      singleChannelFrame.shape, kernelBlurKernel, outlineOnly,
358                                      outlineThickness, negativeHalo, haloThickness, negativeInside).astype(np.int16)
359
360            if np.sum(pMask) > 0:
361                # I square-root the mask sum so that the particles at the edge don't
362                # always win out against actual particles
363                # TODO: This still doesn't give priority to particles on screen;
364                # need to figure out a way to do that
365                particleScores[i] = np.sum(singleChannelFrame * pMask) / np.sqrt(np.sum(pMask))
366            else:
367                particleScores[i] = 0 
368
369        # Sort according to worst scores
370        order = np.argsort(particleScores)[::-1]
371        overlappingParticles = overlappingParticles[order]
372
373        # Now we start removing the particles with the worst scores until we no longer have overlap
374        index = 0
375        while np.sum(adjMat) > 0:
376            adjMat[overlappingParticles[index],:] = 0
377            adjMat[:,overlappingParticles[index]] = 0
378            refinedPeakPositions[overlappingParticles[index]] = None
379            refinedRadii[overlappingParticles[index]] = None
380
381            index += 1
382
383           
384        # Now remove all of the None values we put in
385        refinedPeakPositions = [rpp for rpp in refinedPeakPositions if rpp is not None]
386        refinedRadii = [rr for rr in refinedRadii if rr is not None]
387
388
389    # Debug option
390    if debug:
391        fig = plt.figure(figsize=(12,4))
392
393        ax1 = fig.add_subplot(1, 3, 1, projection='3d')
394        ax1.plot_surface(np.arange(convArr.shape[1]),
395                        np.vstack(np.arange(convArr.shape[0])), convArr, cmap=plt.cm.jet)
396        ax1.set_title('Convolution')
397
398        ax2 = fig.add_subplot(1, 3, 2, projection='3d')
399        ax2.plot_surface(np.arange(convArr.shape[1]),
400                        np.vstack(np.arange(convArr.shape[0])), cleanedConvArr, cmap=plt.cm.jet)
401        ax2.set_title('Cleaned Convolution')
402
403        ax3 = fig.add_subplot(1, 3, 3)
404        ax3.imshow(singleChannelFrame)
405        ax3.set_title('Detected circles')
406        for i in range(len(refinedPeakPositions)):
407            c = plt.Circle(refinedPeakPositions[i][::-1], refinedRadii[i], color='red', fill=False, linewidth=1)
408            ax3.add_artist(c)
409
410        fig.tight_layout()
411
412    return np.array(refinedPeakPositions), np.array(refinedRadii)

415def circularKernelFind(singleChannelFrame, radius, fftPadding, kernelBlurKernel=3, outlineOnly=False, outlineThickness=.05, negativeHalo=False, haloThickness=.03, negativeInside=False, paddingValue=None, debug=False):
416    """
417    Calculate the convolution of a circular mask with an image,
418    identifying likely locations of circles within the image. Adapted from
419    method described in [1].
420
421    To perform the convolution, we take the product of the fft of each the
422    kernel and image, and then ifft the result to transform back to real space.
423    The only peculiarity is that there can be some issues with the edges of image
424    (especially if any circle centers are near the edge/off-screen) so we
425    have to take care to properly pad the image and the kernel.
426
427    Note that this method is not intended to be used as the front end for
428    circle detection for real data. The method `pepe.tracking.convCircle()`
429    makes use of this method to track circles, while also offering many more
430    features. It is recommended to use that function unless there is a very
431    specific purpose that it does not serve.
432
433    Parameters
434    ----------
435    singleChannelFrame : np.ndarray[H,W]
436        A single-channel image (grayscale or just one channel) within which we
437        are looking to identify circular objects.
438
439    radius : float
440        The radius of circles to detect in the image.
441
442    fftPadding : int
443        The amount of padding to add to each side of the image, to prevent the
444        fft from having issues. A good choice is usually 1.5 times the radius,
445        though, less padding can be used if circles are not expected to often
446        be located near the edges.
447
448    kernelBlurKernel : int
449        The kernel size to use to blur the kernel that will be convolved with the image, in
450        pixels. A value of 0 or None will use the original kernel.
451
452    outlineOnly : bool
453        Whether to look for only a particle outline (True), or to look for a filled-in circle (False).
454
455        See `outlineThickness`.
456
457    outlineThickness : float or int
458        Thickness of the particle outline in the kernel; only relevant if `outlineOnly=True`.
459
460        If float, will be taken as fraction of the radius; if integer, will be taken as number
461        of pixels.
462
463    negativeHalo : bool
464        Whether to surround the particle mask with a region of negative value, which
465        further promotes identifying distinct objects as grains.
466
467    haloThickness : float or int
468        Thickness of the negative halo in the kernel; only relevant if `negativeHalo=True`.
469
470        If float, will be taken as fraction of the radius; if integer, will be taken as number
471        of pixels.
472
473    negativeInside : bool
474        Whether to fill the inside of the particle mask with a negative value, which 
475        further promotes identifying outlines of circles.
476
477        Has no effect unless `outlineOnly=True`.
478
479    paddingValue : [float, `'sigmoid'`, or `None`]
480        What value to fill in for the extra padding that is added during the fft/ifft.
481
482        `None` leaves the value at 0.
483
484        `'sigmoid'` will create a smooth-ish transition to 0.
485
486        Any float value will be inserted into every point.
487
488    debug : bool
489        Whether or not to plot various quantities of the calculation for inspection
490        at the end of the evaluation.
491
492    Returns
493    -------
494
495    chiSqr : np.ndarray[H,W]
496        The convolution of the kernel with the image. Larger values represent
497        higher likelihood that there is a particle centered there.
498
499    References
500    ----------
501
502    [1] Franklin, S. V., & Shattuck, M. D. (Eds.). (2016). Handbook
503    of Granular Materials. Chapter 2: Experimental Techniques. p.64-68.
504    CRC Press. https://doi.org/10.1201/b19291
505    """
506
507    if singleChannelFrame.ndim > 2:
508        imageArr = np.mean(singleChannelFrame, axis=-1)
509    else:
510        imageArr = singleChannelFrame
511
512    fftPadding = int(fftPadding)
513
514    # Pad the proper image
515    paddedImageArr = np.zeros((int(imageArr.shape[0] + 2*fftPadding), int(imageArr.shape[1] + 2*fftPadding)))
516    paddedImageArr[fftPadding:-fftPadding,fftPadding:-fftPadding] = imageArr
517
518    # Both dimensions will always have the same first crop index, but possibly a different second one
519    cropBounds = [int(fftPadding + radius), int(-fftPadding + radius), int(-fftPadding + radius)]
520    # If the fftpadding and the radius are exactly the same, we get some weird behavior
521    # This happens because python can index negative values as that element from the end of
522    # the array, but indexing -0 just gives 0, and so it gives you a (0,0) array.
523    if cropBounds[1] >= 0:
524        cropBounds[1] = None
525        cropBounds[2] = None
526
527    # We have a couple options for what values to put in the padding
528    if paddingValue is None or paddingValue == 0:
529        # Do nothing, and leave the value at 0
530        pass
531
532    elif paddingValue == 'sigmoid':
533        # Sigmoid gives a smoothish decay from the image to 0 at the boundaries
534        # Most of the constants here are just arbitrarily chosen to make things look good
535        # Top
536        paddedImageArr[:fftPadding] += np.multiply.outer(2/(1 + np.exp(np.linspace(0, 5, fftPadding))), paddedImageArr[fftPadding])[::-1,:] 
537        # Bottom
538        paddedImageArr[-fftPadding:] += np.multiply.outer(2/(1 + np.exp(np.linspace(0, 5, fftPadding))), paddedImageArr[-fftPadding-1])
539        # Left
540        paddedImageArr[:,:fftPadding] += np.multiply.outer(paddedImageArr[:,fftPadding], 2/(1 + np.exp(np.linspace(0, 5, fftPadding))))[:,::-1]
541        # Right
542        paddedImageArr[:,-fftPadding:] += np.multiply.outer(paddedImageArr[:,-fftPadding-1], 2/(1 + np.exp(np.linspace(0, 5, fftPadding))))
543
544    else:
545        # Just put a fixed value in
546        paddedImageArr[:fftPadding] = paddingValue
547        paddedImageArr[-fftPadding:] = paddingValue
548        paddedImageArr[fftPadding:-fftPadding,:fftPadding] = paddingValue
549        paddedImageArr[fftPadding:-fftPadding,-fftPadding:] = paddingValue
550
551    center = np.array([radius,radius])
552    # To be able to properly convolute the kernel with the image, they have to
553    # be the same size, so just just put our kernel into an array the same size
554    # as the image (though most entries will just be zero)
555    # See function below for more information on generating this kernel
556    kernelArr = genCircularKernel(center, radius, paddedImageArr.shape, kernelBlurKernel, outlineOnly,
557                                 outlineThickness, negativeHalo, haloThickness, negativeInside)
558
559    # First convolutional term
560    convTerm1 = ifft2(fft2(paddedImageArr**2) * fft2(kernelArr))
561    # Trim padding
562    convTerm1 = convTerm1[cropBounds[0]:cropBounds[1],cropBounds[0]:cropBounds[2]]
563
564    # Second term is pretty easy since we choose our weight function to be our
565    # particle mask (and it just has 0s and 1s, so it is equal to it's square)
566    convTerm2 = 2 * ifft2(fft2(paddedImageArr) * fft2(kernelArr))
567    # Trim padding
