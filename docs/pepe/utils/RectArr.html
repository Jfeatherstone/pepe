 10    """
 11    Take the ragged arrays representing particles and forces and convert them to
 12    particle-wise rectangular arrays.
 13
 14    The force input arrays should be 3 dimensional, with indices as follows:
 15
 16        `forceArr[timestep index]
 17                [particle index]
 18                [force index]`
 19
 20    The center array should be 3 dimensional as well, with indices as follows:
 21
 22        `centerArr[timestep index]
 23                 [particle index]
 24                 [coordination (x or y) index]`
 25
 26    The radius array should be 2 dimensional:
 27
 28        `radiusArr[timestep index]
 29                 [particle index]`
 30
 31    The final returned result will be a list of rectangular arrays of the form:
 32
 33        `rectForceArr[particle index]
 34                    [force index]
 35                    [timestep index]`
 36
 37    There are several advantages to restructing like this:
 38
 39        1.  The evolution of a single force can be plotted with something
 40            like `plt.plot(rectForceArr[0][0])`. To do this with the input arrays
 41            would require quite a bit more work since they are lists, and cannot
 42            be sliced.
 43        
 44        2.  While the top-layer index corresponds to a python list, all of the particle-
 45            wise arrays are numpy objects, which allows them to be sliced/saved easily.
 46
 47        3.  Due to potentially having different numbers of particles per frame and different
 48            numbers of forces acting on each particle, the original arrays and very ragged.
 49            This makes any sane indexing very hard to do, which is fixed by reshaping the
 50            arrays. If a force only exists for a subset of the time steps, the rectangular
 51            result will still identify it as a unique force in all timesteps, just with a
 52            magnitude of 0 (or `np.nan`; see `forcePadValue`).
 53
 54
 55    Parameters
 56    ----------
 57
 58    forceArr : list[T,P,F]
 59        Force magnitudes for T timesteps, with P(T) particles per timestep, and F(P(T)) forces
 60        per particle.
 61
 62    alphaArr : list[T,P,F]
 63        Alpha angles for T timesteps, with P(T) particles per timestep, and F(P(T)) forces
 64        per particle.
 65
 66    betaArr : list[T,P,F]
 67        Beta angles for T timesteps, with P(T) particles per timestep, and F(P(T)) forces
 68        per particle.
 69
 70    centerArr : list[T,P,2]
 71        List of particle centers (y,x) for T timesteps, with P(T) particles per timestep.
 72
 73    radiusArr : list[T,P]

 75
 76    angleArr : list[T,P]
 77        List of particle rotations for T timesteps, with P(T) particles per timestep.
 78
 79    maxBetaDisplacement : float
 80        The maximum difference of beta angle between timesteps for which a force will
 81        still be identified with the previous step. The difference is calculated between
 82        the current timestep and whatever the last timestep for which that force existed.
 83
 84    forcePadValue : float
 85        The value that is recorded for a force that isn't current active for a given frame.
 86        Common choices would be `np.nan` -- such that the force won't appear when it is
 87        plotted -- or `0` -- which is a little more physically-inspired.
 88
 89    continuousBetas : bool
 90        When a force is not active for a number of timesteps, should the beta value (position)
 91        of that force be set to `np.nan` (`False`) or should the previous position of the
 92        force be carried over (`True`)? This is the difference between the force not existing
 93        for these timesteps, versus the force existing but having a magnitude of 0.

 95    Returns
 96    -------

100        See description for more info.

104        See description for more info.

108        See description for more info.

113

116        does not exist for a given timestep.
117
118    rectAngleArr : np.ndarray[P,T]
119        Particle angles for each timestep. Elements take on a value of `np.nan` if the particle
120        does not exist for a given timestep.

282        rectBetaArr.append(singleParticleBetaArr)

217            # beta values from the previous frame, since otherwise we will run into
218            # issues later on.
219            if particleIndex < 0:
220                triSortedBetaArr.append(triSortedBetaArr[-1])
221                triSortedForceOrderArr.append([None for i in range(len(triSortedBetaArr[-1]))])
222                continue
223
224            # Correlate the forces between this frame and the last
225            #print(triSortedBetaArr[-1])
226            order = preserveOrderArgsort(triSortedBetaArr[-1], betaArr[j][particleIndex], padMissingValues=True, maxDistance=maxBetaDisplacement, periodic=True)
227            # If we "lose" a force, carry the old beta value through to the new array, so
228            # that we can identify it if it ever pops up again.
229            betaValues = [betaArr[j][particleIndex][order[k]] if order[k] is not None else triSortedBetaArr[-1][k] for k in range(len(order))]
230
231            #print(f'{betaValues}     {order}     {betaArr[j][particleIndex]}')
232
233            triSortedBetaArr.append(betaValues)
234            #triSortedForceOrderArr.append([i for i in order if i is not None])
235            triSortedForceOrderArr.append(order)
236
237        #plt.plot([triSortedBetaArr[j][0] for j in range(len(triSortedBetaArr))])
238        #plt.show()
239
240        # Since the triangular array is only allowed to grow, the length of the last
241        # entry will be the maximum number of *unique* forces.
242        #plt.plot([len(ele) for ele in triSortedBetaArr]) # Check that it only grows
243        #plt.show()
244        maxNumForces = len(triSortedBetaArr[-1])
245
246        if maxNumForces == 0:
247            continue
248
249        # Now we can finally create some nice rectangular arrays
250        singleParticleForceArr = np.zeros((maxNumForces, numTimesteps))
251        singleParticleBetaArr = np.zeros((maxNumForces, numTimesteps))
252        singleParticleAlphaArr = np.zeros((maxNumForces, numTimesteps))
253
254        for j in range(numTimesteps):
255            particleIndex = particleOrder[j,i]
256            
257            # If particle doesn't exist for this frame, fill with nan values and move
258            # on
259            if particleIndex < 0:
260                singleParticleBetaArr[:,j] = np.repeat(np.nan, maxNumForces)
261                singleParticleForceArr[:,j] = np.repeat(forcePadValue, maxNumForces)
262                singleParticleAlphaArr[:,j] = np.repeat(np.nan, maxNumForces)
263                continue
264
265            # We want to remove the carry over beta values for forces that aren't actually there
266            # now, and then pad with nan values for forces that haven't appeared yet
267
268            # Not sure why, but it may be helpful to have the previous beta values carried over,
269            # so that can be done by passing a kwarg
270            if continuousBetas:
271                singleParticleBetaArr[:,j] = [triSortedBetaArr[j][k] for k in range(len(triSortedBetaArr[j]))] + [np.nan]*(maxNumForces - len(triSortedBetaArr[j]))
272            else:
273                singleParticleBetaArr[:,j] = [betaArr[j][particleIndex][triSortedForceOrderArr[j][k]] if triSortedForceOrderArr[j][k] is not None else np.nan for k in range(len(triSortedBetaArr[j]))] + [np.nan]*(maxNumForces - len(triSortedBetaArr[j]))
274
275            # I apologize to anyone who has to read these lines... They do exactly what is stated above,
276            # but boy do they look terrible
277            singleParticleForceArr[:,j] = [forceArr[j][particleIndex][triSortedForceOrderArr[j][k]] if triSortedForceOrderArr[j][k] is not None else forcePadValue for k in range(len(triSortedBetaArr[j]))] + [forcePadValue]*(maxNumForces - len(triSortedBetaArr[j]))
278            singleParticleAlphaArr[:,j] = [alphaArr[j][particleIndex][triSortedForceOrderArr[j][k]] if triSortedForceOrderArr[j][k] is not None else np.nan for k in range(len(triSortedBetaArr[j]))] + [np.nan]*(maxNumForces - len(triSortedBetaArr[j]))
279
280        # Append all of the nice rectangular arrays
281        rectForceArr.append(singleParticleForceArr)
282        rectAlphaArr.append(singleParticleAlphaArr)
283        rectBetaArr.append(singleParticleBetaArr)
284
285    return rectForceArr, rectAlphaArr, rectBetaArr, rectCenterArr, rectRadiusArr

Take the ragged arrays representing particles and forces and convert them to particle-wise rectangular arrays.

The force input arrays should be 3 dimensional, with indices as follows:

`forceArr[timestep index]
        [particle index]
        [force index]`

The center array should be 3 dimensional as well, with indices as follows:

`centerArr[timestep index]
         [particle index]
         [coordination (x or y) index]`

The radius array should be 2 dimensional:

`radiusArr[timestep index]
         [particle index]`

The final returned result will be a list of rectangular arrays of the form:

`rectForceArr[particle index]
            [force index]
            [timestep index]`

There are several advantages to restructing like this:

1.  The evolution of a single force can be plotted with something
    like `plt.plot(rectForceArr[0][0])`. To do this with the input arrays
    would require quite a bit more work since they are lists, and cannot
    be sliced.

2.  While the top-layer index corresponds to a python list, all of the particle-
    wise arrays are numpy objects, which allows them to be sliced/saved easily.

3.  Due to potentially having different numbers of particles per frame and different
    numbers of forces acting on each particle, the original arrays and very ragged.
    This makes any sane indexing very hard to do, which is fixed by reshaping the
    arrays. If a force only exists for a subset of the time steps, the rectangular
    result will still identify it as a unique force in all timesteps, just with a
    magnitude of 0 (or `np.nan`; see `forcePadValue`).
Parameters
  • forceArr (list[T,P,F]): Force magnitudes for T timesteps, with P(T) particles per timestep, and F(P(T)) forces per particle.
  • alphaArr (list[T,P,F]): Alpha angles for T timesteps, with P(T) particles per timestep, and F(P(T)) forces per particle.
  • betaArr (list[T,P,F]): Beta angles for T timesteps, with P(T) particles per timestep, and F(P(T)) forces per particle.
  • centerArr (list[T,P,2]): List of particle centers (y,x) for T timesteps, with P(T) particles per timestep.
  • radiusArr (list[T,P]): List of particle radii for T timesteps, with P(T) particles per timestep.
  • angleArr (list[T,P]): List of particle rotations for T timesteps, with P(T) particles per timestep.
  • maxBetaDisplacement (float): The maximum difference of beta angle between timesteps for which a force will still be identified with the previous step. The difference is calculated between the current timestep and whatever the last timestep for which that force existed.
  • forcePadValue (float): The value that is recorded for a force that isn't current active for a given frame. Common choices would be np.nan -- such that the force won't appear when it is plotted -- or 0 -- which is a little more physically-inspired.
  • continuousBetas (bool): When a force is not active for a number of timesteps, should the beta value (position) of that force be set to np.nan (False) or should the previous position of the force be carried over (True)? This is the difference between the force not existing for these timesteps, versus the force existing but having a magnitude of 0.
Returns
  • rectForceArr (listP): A list of arrays representing the force magnitude for each force on each particle. See description for more info.
  • rectAlphaArr (listP): A list of arrays representing the alpha angle for each force on each particle. See description for more info.
  • rectBetaArr (listP): A list of arrays representing the beta angle for force on each particle. See description for more info.
  • rectCenterArr (np.ndarray[P,T,2]): Particle centers for each timestep. Elements take on a value of [np.nan, np.nan] if the particle does not exist for a given timestep.
  • rectRadiusArr (np.ndarray[P,T]): Particle radii for each timestep. Elements take on a value of np.nan if the particle does not exist for a given timestep.
  • rectAngleArr (np.ndarray[P,T]): Particle angles for each timestep. Elements take on a value of np.nan if the particle does not exist for a given timestep.
